# Ограничения и принципы для агентов

Выдержка из `docs/nauchny_klubok_top1_tz.md` §8, §35, §37.

## Продуктовые принципы

1. **Evidence-first** — confirmed факт только с источником и SourceSpan; иначе candidate.
2. **Claim-based** — не абсолютная истина в графе, а claims с provenance.
3. **Открытая схема** — предметные типы из реестра, не хардкод в коде.
4. **Гибридный поиск** — не только embeddings: граф, лексика, таблицы, алиасы, числа.
5. **Query IR** — не свободный Cypher от LLM как основной путь.
6. **Локальный граф ответа** — компактный доказательный subgraph, не весь граф.
7. **Access до синтеза** — retrieval с учётом роли до генерации ответа.
8. **Тяжёлое на ingestion** — не в момент каждого запроса.

## Правила работы агента

- Одна конкретная задача с входными/выходными DTO и Definition of Done.
- Не «сделай весь сервис».
- Можно: boilerplate, тесты, DTO, адаптеры, prompts, миграции, Dockerfile.
- Нельзя без решения команды: shared contracts, ontology, domain pack, Docker networking, public API, миграции, security rules.
- Нельзя доверять агенту: доменную истинность, финальные answers, aggressive entity merge, security policy.

## Запреты

- Чат по документам без доказательств.
- Ответы без источников; хардкод на официальные вопросы.
- Полный граф на миллион узлов; свободный Cypher без Query IR.
- Игнорировать географию, роли, аудит.
- Менять контракты без sync; фичи после freeze.
- OWL/OCR как обязательный путь, если текст и таблицы уже доступны.

## Дисциплина

Сначала MVP end-to-end, потом top-1. Ни одного подтверждённого факта без источника.
