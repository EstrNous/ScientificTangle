# Полные правила агентов (L1)

Расширение `AGENTS.md`. Читать при старте задачи, если `task_router.md` указывает на этот файл.

## Общие правила разработки

- Главный ориентир — `docs/nauchny_klubok_top1_tz.md` и выдержки в `docs/tz/`; решения не должны противоречить MVP и demo-сценарию.
- Перед имплементацией обдумай архитектурное место, план, риски и критерии проверки.
- Перед завершением проверь соответствие Definition of Done из ТЗ или явному запросу пользователя.
- Если меняется общее правило — обнови `AGENTS.md`, при необходимости `docs/agent_prompts/` и `docs/agent_context/`. Адаптеры платформ не дублируют правила.

## Git и ветки

Полный workflow: `docs/agent_context/git_workflow.md`.

Кратко:

- По умолчанию работай в `feat/*`, созданной от актуального `origin/dev`.
- Синхронизация: `git fetch origin`, затем `git rebase origin/dev` на feat-ветке.
- **Не мержи локально в `dev`**. Интеграция — push `feat/*` + PR → merge на GitHub.
- Локальная проверка совместимости — throwaway-ветка или worktree, без изменения локального `dev`.
- Перед рискованным rebase: `git branch backup/<ветка>-<дата> HEAD`.
- Конфликты: разбирать вручную по файлам, тесты после; при сомнении — abort и спросить пользователя.
- Если в `main` — переключись на `dev`, создай `feat/*`.
- Перед коммитом: `git status`, без лишних файлов.

## Коммуникация

- Кратко, по делу, на русском, если пользователь не попросил иначе.
- Если задача стала другой — предложи новый чат с промптом из `docs/agent_prompts/new_chat.md`.
- Если контекст перегружен — новый чат с кратким summary состояния.
- Предупреждай о рисках: локальный merge в `dev`, force-push в shared-ветки, push в `main`, рассинхрон правил, случайное изменение `README.md`.

## Импорты Python

В runtime-коде пакета `services/<name>/app/`:

- локальные модули — `from .module`, `from ..subpackage.module`;
- `shared.*` — абсолютный импорт как отдельная path-зависимость, не как локальный модуль сервиса;
- stdlib и PyPI — абсолютные импорты.

В тестах сервиса допустим `from app.*` при запуске с `PYTHONPATH`, указывающим на корень сервиса.

Запрещены `from app.*` внутри `services/<name>/app/` (кроме legacy до миграции на relative imports).
