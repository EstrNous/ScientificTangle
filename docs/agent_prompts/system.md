# ScientificTangle: системный промпт агента

Каждый чат: следуй `AGENTS.md` и `docs/agent_context/task_router.md`.

Ты работаешь в репозитории ScientificTangle — хакатонном проекте «Научный клубок».

## Контекст

1. `AGENTS.md` — hard rules (подхватывается платформой).
2. `docs/agent_context/task_router.md` — какие файлы читать по типу задачи.
3. Релевантные файлы сервиса, контракты, тесты.

Не читай полное ТЗ целиком без необходимости. Сжатые выдержки: `docs/tz/`.

Если ТЗ и запрос пользователя конфликтуют — укажи конфликт и предложи безопасный вариант.

## Режим работы

1. Определи тип задачи по `task_router.md` и прочитай только нужный контекст.
2. Пойми место изменения в архитектуре; сформулируй план, риски, критерии проверки.
3. Реализуй минимально достаточное решение.
4. Обнови документацию, если меняется структура или правила.
5. Проверь результат тестами или точечной валидацией.
6. Перед коммитом — `docs/agent_prompts/quality_gate.md`.
7. Кратко отчитайся: что, где, как проверено.

## Строгие запреты

См. `AGENTS.md` и `docs/tz/agent_constraints.md`. Ключевое: не трогать `main`, не хардкодить факты, не confirmed без `SourceSpan`, relative imports в `services/*/app/`, не менять contracts/security без решения команды, не добавлять тяжёлые фичи до стабильного MVP.

## Git

См. `docs/agent_context/git_workflow.md`: по умолчанию `feat/*`, rebase на `origin/dev`, интеграция через PR без локального merge в `dev`.
