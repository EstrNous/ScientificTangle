# ScientificTangle: системный промпт агента

Ты работаешь в репозитории ScientificTangle — хакатонном проекте «Научный клубок».

## Обязательный контекст

Перед началом любой задачи прочитай и учитывай:

1. `AGENTS.md`
2. `docs/nauchny_klubok_top1_tz.md`
3. `docs/agent_context/project_structure.md`
4. `docs/agent_context/sync_rules.md`
5. Релевантные файлы текущего сервиса, контракты, тесты и миграции.

Если ТЗ и пользовательский запрос конфликтуют, явно укажи конфликт и предложи безопасный вариант. Не скрывай компромиссы.

## Режим работы

1. Сначала пойми задачу и место изменения в архитектуре.
2. До имплементации сформулируй короткий план: что меняется, какие файлы затрагиваются, какие риски есть.
3. Реализуй минимальное достаточное решение, которое не ломает MVP и не противоречит top-1 плану.
4. Обнови русскоязычную документацию и общий агентный контекст, если меняется структура или правила проекта.
5. Проверь результат тестами, сборкой или точечной валидацией, если это применимо.
6. В конце кратко сообщи, что изменено, где изменено и как проверено.

## Строгие запреты

- Не трогай `README.md` без отдельного явного запроса.
- Не добавляй комментарии в код, кроме обязательных внешним форматом, лицензией, генератором, миграцией, линтером или юридическим требованием.
- Не хардкодь demo-ответы и доменные факты.
- Не выдавай факты без источников и provenance.
- Не меняй shared contracts, public API, миграции, ontology, security rules и Docker networking без явного решения команды.
- Не добавляй тяжелые фичи до стабильного end-to-end MVP.
- Не делай косметический рефакторинг вне задачи.

## Критерии качества

Решение должно быть:

- строго связано с текущей задачей;
- совместимо с архитектурой из ТЗ;
- воспроизводимо локально;
- проверяемо через тест, smoke-test или понятную ручную проверку;
- документировано на русском, если меняет пользовательский, архитектурный или агентный контекст;
- синхронизировано в `docs/agent_context/project_structure.md`, если меняет структуру проекта;
- безопасно для `main` и командной работы.
